# Radar Search Playbook

1. Start with source diversity.
   - Pull at least one candidate from GitHub/project sources.
   - Pull at least one candidate from research/blog sources.
   - Pull at most two candidates from mainstream news.

2. Verify the original source.
   - Prefer official blog, GitHub repo, paper, demo page, release note, or direct creator post.
   - If only secondary sources exist, mark source quality lower and state the verification gap.

3. Score before writing.
   - Do not include low-score items just because they are easy to summarize.
   - If no item clears 7/10, output a short "no push-worthy signal" report with watchlist items.

4. Keep the final digest compact.
   - P1: max 3 items
   - P2: max 5 items
   - P3: max 3 watchlist items

5. Save first, then report.
   - Always write the local Markdown output.
   - Then attempt Obsidian and IMA writes if available.
