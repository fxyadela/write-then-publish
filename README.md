# 写了就发（Write Then Publish）

一个面向内容创作者的本地排版编辑器。
输入一段内容后，可以在「图文卡片」和「长文」之间一键切换：图文自动分页导出图片，长文按文章样式预览排版。

这是基于 [fxyadela/write-then-publish](https://github.com/fxyadela/write-then-publish) 的个人改版，重点增强了长稿编辑体验、本地数据可靠性、跨浏览器使用和高清导出。

它解决的不是“写作”本身，而是写完之后最麻烦的部分：排版、分版、预览、下载、历史保存和多格式复用。

## 核心功能

- `图文卡片模式`：长文本自动分页，生成适合小红书 / X 风格发布的竖版图片。
- `长文模式`：支持 Markdown 长文预览，提供主题、字体、字号和主题色设置。
- `一键互转`：图文可以转长文，长文也可以自动分页转成图文卡片；内容没写完也能转换。
- `滑动截图模式`：在单张长卡片内滚动查看内容，适合手动截图。
- `历史记录`：自动保存草稿，支持新建、恢复、删除。
- `自定义项目标题`：标题可独立于正文修改，历史记录与 JSON 备份会保留自定义名称。
- `双层本地保存`：正文短延迟写入 LocalStorage，IndexedDB 合并写入；大图片较多时仍有本地数据库副本，又不会阻塞每次输入。
- `正文快速保存`：文字和图片引用使用轻量增量写入；删掉图片引用时立即重绘并落盘，不必反复写入整份大图数据库。
- `流畅实时预览`：输入会合并触发重绘，已解码图片和未变化卡片会直接复用，减少多页图文的全量 Canvas 重画。
- `预览定位锁定`：悬浮右侧文字可快速定位左侧原文；点击后会锁定编辑位置，横向浏览其他卡片时不再被悬浮内容带着跳动。
- `跨浏览器共享草稿`：本地服务把草稿同步保存到磁盘共享库，Chrome 与 Codex 内置浏览器刷新后可读取同一版本；浏览器 IndexedDB 继续作为离线兜底。
- `草稿备份与导入 JSON`：可将全部草稿、内嵌图片和排版设置导出为 JSON；在其他地址或浏览器选择本地 JSON 后，会先预览来源、文章和新增/更新数量，再合并到当前草稿库。
- `历史分类筛选`：历史记录自动标记为「图文」或「长文」，可按类型筛选。
- `个人信息定制`：支持修改头像、名称、昵称，并可选择每页显示头像或仅首页显示。
- `Obsidian 仓库同步`：首次授权本地 Obsidian 仓库后，直接把 Obsidian Markdown 粘贴到编辑区，系统会自动识别 `![[图片.png]]` 与 `![](附件/图片.png)` 并读取本机附件。
- `快捷放图`：可在正文编辑框直接粘贴截图、复制的图片或拖入图片文件。
- `图片处理`：支持正文插图、多图上传、头像裁剪、图片位置/大小调整和批量宽度设置。
- `文字样式`：支持大标题、小标题、加粗、斜体、引用、字体颜色、文字背景色。
- `样式互通`：图文卡片和长文共用字体颜色、文字背景色等行内样式语法。
- `手动留白`：图文卡片会保留编辑框中的回车空行，方便手动控制节奏。
- `行距控制`：图文卡片行距最低可调到 `1.0`，用于更紧凑的文字排版。
- `段间距控制`：正文段落不会额外叠加过大的段前段后空隙，手动空行按接近小红书图文的段落间隔显示。
- `字体设置`：支持中文字体和英文字体分别选择。
- `查找替换`：支持查找、替换当前、全文替换。
- `导出下载`：支持单张下载和批量打包下载。
- `高清导出`：卡片支持 `1× / 2× / 3×` 重新绘制导出，保持 `3:4` 比例；默认 `2×` 为 `1728x2304`。
- `长文下载`：长文模式支持导出为 PNG 长图，保留当前排版样式。
- `编辑撤销`：支持 `Command + Z` 撤销和 `Command + Shift + Z` 重做。
- `黑白主题`：界面按钮、卡片背景和字体颜色会随主题自适应。

## 两种工作区

### 图文卡片

适合把长文拆成多张图片发布。系统会根据页面容量自动切割内容，并尽量保留标题、引用、加粗、颜色、图片等样式。

适用场景：

- 小红书图文
- X 长帖截图
- 知识卡片
- 长文拆条
- 图文教程

### 长文

适合先把内容整理成文章样式。支持 Markdown 标题、引用、代码块、图片和基础行文结构，并提供独立的长文样式设置。

适用场景：

- 公众号文章草稿
- 长文预览
- Markdown 排版
- 图文内容转长文沉淀

## 一键互转

顶部的转换按钮会根据当前模式自动变化：

- 在图文模式下：显示 `转长文`
- 在长文模式下：显示 `转图文`

转换不会重写内容，只会切换排版输出方式：

- 图文转长文：当前编辑内容会进入长文预览。
- 长文转图文：当前编辑内容会自动分页生成图文卡片。

这意味着你可以先随便写，不必一开始就决定最终发布格式。

## 历史记录

左侧历史栏会自动保存当前草稿，并支持：

- 点击恢复历史内容
- 删除不需要的记录
- 按 `全部 / 图文 / 长文` 筛选
- 自动显示每条记录的类型标签

历史记录会保存在浏览器本地存储中。通过 `npm start` 使用主端口时，还会同步写入 `runtime-data/project-store.json`，让 Chrome 与 Codex 内置浏览器在刷新后读取同一份较新草稿。

为了避免清理网站数据或更换浏览器后丢失，建议定期点击顶部“备份”，下载包含全部草稿、图片和排版设置的 JSON。需要跨 `127.0.0.1`、`localhost` 或不同端口迁移时，在原地址导出，在主端口点击“导入 JSON”，确认预览后合并即可；其他已有草稿不会被删除，同一 ID 只保留时间更新的版本。

包含较多图片时，草稿会自动写入 IndexedDB；点击“备份”会先强制完成数据库落盘，再下载 JSON，避免刚改完就刷新或关闭页面时回退到旧快照。

本地自动恢复可使用同源参数 `?restoreBackup=/backups/文件名.json`。恢复完成后参数会自动移除；跨域地址会被拒绝，避免误导入外部内容。

## 本地使用

这个项目是纯前端静态页面，可以直接打开 `index.html` 使用。

也可以在项目目录执行：

```bash
npm run open
```

或启动本地服务：

```bash
npm start
```

然后访问：

```text
http://127.0.0.1:5173/
```

## 使用流程

1. 在左侧编辑框输入或粘贴内容。
2. 选择 `图文卡片` 或 `长文` 工作区。
3. 用工具栏调整标题、加粗、斜体、引用、颜色、背景色和图片。
4. 修改头像、名称、昵称和头像显示方式。
5. 需要留白时，在编辑框里直接回车空行，右侧图文会保留这些空白行。
6. 在右侧预览区检查排版。
7. 需要复用时点击 `转长文` 或 `转图文`。
8. 图文模式下可单张下载或批量下载。

## 文字上色

点击画笔按钮后：

1. 选择字体颜色。
2. 点击 `确定`。
3. 在编辑框中选中需要上色的文字。

工具会自动给选中文字添加颜色标记，并刷新右侧预览。文字背景色也是同样逻辑。

## 图片与头像

- 头像支持上传和裁剪。
- 正文图片支持插入、多图上传、裁剪、调整大小和对齐。
- 图片面板支持输入宽度百分比后批量应用，图片会按原比例等比缩放。
- 图片面板支持批量设置固定宽高；固定框会按原比例居中裁切，不会把图片拉变形。
- 图片默认尽量保留原比例，不主动压缩画质。
- 如果图片已经裁好，可以直接上传使用。

## 本地改版说明

这个 fork 在原版排版能力上补充了：

- 多篇草稿的 JSON 备份、预览导入和安全合并。
- IndexedDB 大图持久化及刷新恢复顺序修复。
- 磁盘共享草稿库，解决同端口不同浏览器缓存不互通的问题。
- 延迟保存、Canvas 与图片缓存，降低长稿实时预览的重绘压力。
- 1× / 2× / 3× 等比例高清导出。
- 预览文字定位与点击锁定，方便在多卡片长稿中稳定修改原文。

## 技术说明

- 前端仍可静态部署；本地 `npm start` 会额外启动轻量 Python 服务，用于跨浏览器共享草稿。
- 图文卡片使用 Canvas 渲染。
- 批量下载使用 JSZip 在浏览器内打包。
- 草稿和历史记录使用浏览器本地存储保存。
- 部署可直接使用 Vercel 静态站点配置。
